# My Application

The project is generated by [LoopBack](http://loopback.io).

## Step 1: Generate MongoDB Embeddings (AI Preparation)

This repo includes a Python script to generate embeddings from *all fields* in
`case` and `legislation` collections, then store vectors back in MongoDB.

### 1) Install Python dependencies

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r scripts/requirements-embeddings.txt
```

### 2) Run embeddings for both collections

```bash
python scripts/generate_embeddings.py \
  --mongo-uri "$(cat MONGODB_CONNECTION_URL)" \
  --db-name apptorney
```

By default this writes:
- `case.caseEmbedding`
- `legislation.legislationEmbedding`

### Optional flags

- `--dry-run`: compute but do not write
- `--include-existing`: re-embed documents that already have embedding fields
- `--limit 100`: process only first 100 docs per collection
- `--batch-size 16`: lower memory usage when embedding large text

## Vector Search API Endpoints

The backend now exposes three vector endpoints:

- `POST /api/cases/vector-search`
- `POST /api/legislations/vector-search`
- `POST /api/search/universal-vector-search`
- `POST /api/search/ask-ai`

Request body (either provide `queryVector` or `queryText`):

```json
{
  "queryText": "unlawful termination and compensation",
  "limit": 10,
  "maxCandidates": 2000
}
```

If you already have a vector:

```json
{
  "queryVector": [0.01, -0.03, 0.44],
  "limit": 10
}
```

Optional body flags:

- `includeDeleted` (boolean): include soft-deleted records
- `includeCases` (boolean): universal endpoint only, default `true`
- `includeLegislations` (boolean): universal endpoint only, default `true`

### AI natural response endpoint

`POST /api/search/ask-ai`

```json
{
  "question": "What is the legal position on wrongful termination?",
  "retrievalLimit": 8,
  "maxCandidates": 2000
}
```

Required environment variables:

- `OPENAI_API_KEY`

Optional environment variables:

- `OPENAI_CHAT_MODEL` (default: `gpt-4o-mini`)
- `EMBEDDING_PYTHON_BIN` (example: `.venv/bin/python`)
- `EMBEDDING_MODEL` (default: `sentence-transformers/all-MiniLM-L6-v2`)
- `EMBEDDING_QUERY_SCRIPT` (default: `scripts/embed_text.py`)

`ask-ai` retrieves top vector matches from cases/legislations, then asks OpenAI to
generate a grounded natural-language answer with inline source tags like `[S1]`.

### Container readiness

The Docker image is configured to run both Node.js and Python embedding scripts:
- Installs Python 3 + pip
- Installs `scripts/requirements-embeddings.txt`
- Sets `EMBEDDING_PYTHON_BIN=/usr/bin/python3`
- Sets `EMBEDDING_QUERY_SCRIPT=/usr/src/app/scripts/embed_text.py`

You still need to inject `OPENAI_API_KEY` at runtime.

## Alternative: Voyage Law 2 Embeddings

If you want a separate legal embedding space, use:

- `scripts/generate_embeddings_voyage.py`
- `scripts/requirements-voyage-embeddings.txt`

This writes new fields (without replacing existing ones):
- `case.caseEmbeddingVoyage`
- `legislation.legislationEmbeddingVoyage`

Setup and run:

```bash
source .venv/bin/activate
pip install -r scripts/requirements-voyage-embeddings.txt

export VOYAGE_API_KEY="your-voyage-api-key"
python scripts/generate_embeddings_voyage.py \
```
